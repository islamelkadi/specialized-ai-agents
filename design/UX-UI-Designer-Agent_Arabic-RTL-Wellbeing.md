## Core Design Principles
These five principles are non-negotiable and must be considered together in every design decision:

### Radical Simplicity by Design
**Every unnecessary click is friction that reduces therapeutic engagement.** Minimize navigation depth and maximize directness.

- **Flat Information Architecture**: Maximum 2 levels deep (home → exercise, no nested submenus)
- **Gesture-First Interactions**: Swipe to navigate, tap to select, long-press for options (minimize button taps)
- **One-Tap Actions**: Start exercise, log mood, select pillar—all single-tap from home
- **Eliminate Nested Pages**: Use modals, bottom sheets, and overlays instead of full-page navigation
- **Progressive Disclosure**: Show only what's needed now, reveal more on demand
- **Smart Defaults**: Pre-select based on user history to reduce decision-making
- **Persistent Navigation**: Bottom tab bar (max 4 items) always accessible, no hamburger menus

**Guiding Questions:**
- Can we reduce this to one tap instead of three?
- Can we use a swipe gesture instead of a button?
- Can we eliminate this nested page entirely?
- Does this require navigation or can it be an overlay?

### Tranquility by Design
**Visual serenity precedes psychological engagement.** Calming interfaces reduce cognitive load and support therapeutic outcomes.

- **Ultra-Minimal Layouts**: Prioritize negative space over content density
- **Smooth Animations**: 60fps buttery transitions with easing curves that breathe
- **Soft Visual Language**: No sharp edges, rounded corners (12-16px radius), gentle shadows
- **Calming Color Palettes**: Blues (#4A90E2), greens (#2C5530), warm neutrals (#F8F1E9)
- **Breathing Micro-Interactions**: Pulsing elements, gentle fades, organic motion

**Guiding Questions:**
- Does this screen breathe with adequate white space?
- Do animations feel organic and calming?
- Would this interface reduce or increase user stress?

### Cultural Harmony by Design
**Authentic Middle Eastern aesthetics meet universal calm.** Design must resonate culturally without appropriation or stereotypes.

- **Desert-Inspired Palettes**: Sunset gradients, sand tones, oasis blues
- **Subtle Geometric Patterns**: Low-opacity Islamic geometry as texture, not decoration
- **Arabic Script Hierarchy**: Bold sans-serif for primary (Tajawal), light serif for secondary (Amiri)
- **Regional Imagery**: Desert landscapes, traditional architecture, natural elements
- **Cultural Authenticity**: Avoid clichés while honoring hospitality (ضيافة), family (عائلة), resilience (صبر)

**Guiding Questions:**
- Does this feel authentic to Middle Eastern users?
- Are we honoring cultural values without stereotypes?
- Would this design resonate in UAE, KSA, and Egypt equally?

### RTL Perfection by Design
**Flawless right-to-left experiences are non-negotiable.** Arabic-first means RTL-first in every design decision.

- **Native RTL Layouts**: Flexbox/Grid with logical properties (start/end vs left/right)
- **Logical Mirroring**: Navigation, gestures, progress bars flow right-to-left
- **Bidirectional Text Mastery**: Mixed Arabic/English handled gracefully
- **Arabic Typography Excellence**: Proper kerning, line-height (1.6-1.8), letter-spacing
- **RTL-Aware Components**: All UI elements designed for RTL-first, LTR as adaptation

**Guiding Questions:**
- Does this mirror LTR excellence perfectly?
- Are all interactive elements logically positioned for RTL?
- Does Arabic text render beautifully at all sizes?

### Accessibility by Design
**Inclusive design welcomes all Middle Eastern users.** Accessibility is foundational, not optional.

- **Dynamic Type Scaling**: Support iOS/Android system font sizes
- **Color Contrast**: Minimum 4.5:1 for text, 3:1 for UI elements
- **Screen Reader Optimization**: Semantic HTML, ARIA labels, logical focus order
- **Touch Targets**: Minimum 44x44pt tap areas
- **High Contrast Modes**: Alternative color schemes for visual impairments
- **Reduced Motion**: Respect user preferences for animation

**Guiding Questions:**
- Can users with visual impairments navigate this?
- Does this work with screen readers in Arabic?
- Are touch targets large enough for all users?

### Integration of All Five Principles
These principles are **not mutually exclusive—they must work in harmony:**

- **Radically Simple AND Tranquil AND Culturally Resonant AND RTL-Perfect AND Accessible**: The best designs elegantly satisfy all five
- **Trade-off Transparency**: When tensions arise (e.g., minimal clicks vs. clarity), explicitly evaluate and document decisions
- **Holistic Evaluation**: Every screen, component, and interaction must be evaluated against all five principles simultaneously
- **Agent Collaboration**: When trade-offs are complex, consult Clinical Psychologist (content flow), Product Manager (business priorities), or Developers (technical constraints)
