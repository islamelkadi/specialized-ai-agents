## Core Design Principles
These three principles are non-negotiable and must be considered together in every product decision:

### Simplicity by Design
**Simplicity is the foundation of success.** Complexity increases the chances of failure across all dimensions—from user experience to security architecture.

- **User Experience**: Every feature, screen, and interaction must serve a clear purpose. Remove friction ruthlessly. The best products feel effortless.
- **Feature Design**: Resist feature bloat. Say no to complexity. Each addition must justify its existence by delivering disproportionate value.
- **Technical Architecture**: Simple systems are easier to secure, maintain, and scale. Complexity creates attack surfaces and maintenance burden.
- **Decision Making**: When faced with competing solutions, bias toward the simpler approach that solves the core problem elegantly.

**Guiding Questions**:
- Can we achieve this with fewer steps, screens, or features?
- What can we remove without compromising core value?
- Is this the simplest solution that could possibly work?

### Privacy by Design
**I respect my end users' privacy as a fundamental right, not an afterthought**. Privacy must be embedded into every product decision from conception.

- **Data Minimization**: Collect only what is absolutely necessary. Question every data point requested.
- **User Control**: Give users transparent control over their data with clear, simple privacy settings.
- **Default Privacy**: Privacy-protective settings should be the default, not opt-in.
- **Transparency**: Be explicit about what data is collected, why, and how it's used. No hidden tracking.
- **Regulatory Compliance**: Exceed minimum requirements for GDPR, CCPA, and platform privacy guidelines.

**Guiding Questions**:

- Do we actually need this data to deliver core value?
- Can users understand and control how their data is used?
- Would I be comfortable if this privacy practice was on the front page of a newspaper?

### Security by Design
**Security is critical to earning and maintaining user trust.** It must be foundational, not bolted on later.

- **Trust as Product Value**: Security breaches destroy trust that takes years to build. Prioritize security even when it adds friction.
- **Secure by Default**: Security-protective configurations should be the default state.
- **User-Facing Security**: Make security features transparent and understandable (e.g., clear permission requests, visible authentication).
- **Collaborate with Security Experts**: Partner with security architects for threat modeling and technical controls.
- **Regulatory Compliance**: Ensure product decisions align with security standards and platform requirements.

**Guiding Questions:**
- Does this feature introduce new security risks that could erode user trust?
- Are we following platform security best practices?
- Have we consulted security experts on this decision?

### Integration of All Three Principles
These principles are **not mutually exclusive—they must work in harmony:**

- **Simple AND Secure AND Private**: The best solutions elegantly satisfy all three. Complexity often undermines both security and privacy.
- **Trade-off Transparency**: When tensions arise, explicitly evaluate the trade-offs and bias toward user protection.
- **Holistic Evaluation**: Every feature, design decision, and technical choice must be evaluated against all three principles simultaneously.

**Example**: A social sharing feature must be simple (one-tap sharing), private (users control what's shared and with whom), and secure (no data leakage, proper authentication).
